注解

注解

是什么

以@interface方式定义的接口，也称元数据，与class，interface，enum类型属于同一级别

有什么用

  • 给类配置一些元数据
  • 代码编译前语法检查

使用场景

java框架，如spring，springmvc

需要给类添加一些元数据供使用中获取，如作者信息，需要的依赖等

常用注解

@Override

@Deprecated

@SuppressWarnings

@Inherited

允许子类继承父类的注解

@Documented

指示编译器将被注解的元素的注释信息包含在生成的API文档中

@Retention

指定注解的最大生命周期，表示注解在什么运行级别前有效

参数类型为RetentionPolicy枚举类型的值
RetentionPolicy value();

RetentionPolicy.SOURCE  表示在源码期间有效，可用于代码的语法检查
RetentionPolicy.CLASS	表示注解在源码和编译期间有效，若注解类不配置@Retention，则默认使用该级别
RetentionPolicy.RUNTIME		表示注解在源码，编译，运行时均有效

详见：java.lang.annotation.RetentionPolicy

@Target

指定注解声明的位置，如类上，方法上，字段上

参数类型为枚举类型的数组
ElementType[] value();
如：
ElementType.TYPE		声明在类或接口上
ElementType.FIELD		声明在字段上
ElementType.METHOD		声明在方法上
ElementType.PARAMETER	声明在参数上
ElementType.CONSTRUCTOR
ElementType.LOCAL_VARIABLE
ElementType.ANNOTATION_TYPE		声明在注解类上

详见：java.lang.annotation.ElementType

注解的使用

创建注解

public @interface 注解类名 {}

创建注解的参数(可选)

参数格式：
参数类型 参数名();
参数类型 参数名() default 默认参数值;

如：
String[] name();
String name() default "Tom";

配置注解的注解(可选)

@Target：指定该注解类声明的位置，如在类上声明，方法上声明，字段上声明等

@Retention

@Documented

反射获取注解的参数值

  • 获取使用了注解的类
  • 获取注解
  • 获取注解的值
Class<Student1> clz = Student1.class;
Student annotation = clz.getAnnotation(Student.class);
String name = annotation.name();
int age = annotation.age();
String[] hobby = annotation.hobby();

demo

Author.java

import java.lang.annotation.*;

@Retention(value = RetentionPolicy.RUNTIME)
@Target( { ElementType.TYPE } )
@Inherited
public @interface Author {

    String name() default "your_name";
    int age() default -1;
    String profile() default "<https://www.example.com>";
}
