JavaWeb理论

MVC设计模式

MVC模式（Model-View-Controller）是软件工程中的一种软件架构模式，把软件系统分为三个基本部分。

Model称为模型，也就是实体类。用于数据的封装和数据的传输。如Service、Dao、JavaBean等

View为视图，也就是GUI组件，用于数据的展示。采用的技术有HTML、JSP、前端框架等

Controller为控制器，也就是事件，用于事件流程的控制。采用的技术有Servlet、Filter等

三层架构

业务层

也叫逻辑层，service层。采用事务脚本模式，将一个业务中所有的操作封装成一个方法，同时保证方法中所有的数据库更新操作，即保证同时成功或同时失败。避免部分成功部分失败引起的数据混乱操作。

表现层

也叫web层。包含JSP，Servlet等web相关的内容。

持久层

也叫数据层。采用DAO模式，建立实体类和数据库表映射（ORM映射）。也就是哪个类对应哪个表，哪个属性对应哪个列。持久层的目的是完成对象数据和关系数据的转换。

主流开发框架

SSH框架

  • Spring——业务层
  • Struts——表现层
  • Hibernate——持久层

SSM框架

  • Spring——业务层
  • SpringMVC——表现层
  • MyBatis——持久层

三层架构与MVC模式的关系

两者并无直接的关联。

web层（表现层）相当于MVC中的View。

service层和dao层相当于MVC中的Model。

Filter

基础知识

功能

拦截请求，对sevlet要处理的数据做一些前置处理

概念

Filter对象：由servlet容器创建，filter对象是实现了Filter接口或继承了其实现类的java类

FilterConfig：由servlet容器创建，在servlet容器初始化一个filter对象时，作为参数传递，以获取filter的信息

Filter接口

public interface Filter {

    default public void init(FilterConfig filterConfig) throws ServletException {}

    public void doFilter(ServletRequest request, ServletResponse response,FilterChain chain)
            throws IOException, ServletException;

    default public void destroy() {}
}

FilterAPI

方法名 功能 调用
init 加载Filter的配置信息（filter类名，filter映射等） 在web容器（tomcat）创建filter对象后调用
doFilter 在servlet处理请求之前执行一些操作的方法
destroy 销毁filter 停止web容器前调用

Servlet

基础知识

功能

接收客户端发送过来的请求进行处理，并响应数据给客户端

概念

web容器/servlet容器：运行web应用的服务器。一般指tomcat等

servlet对象：由servlet容器创建，servlet对象是实现了Servlet接口或继承了其实现类的java类

ServletContext：由servlet容器创建，用于servlet对象与servlet容器进行通信，servlet对象可以获取servlet容器中的信息资源。

Servlet容器在启动一个Web应用时，会为它创建一个ServletContext对象。每个web应用有唯一的ServletContext对象。同一个web应用的所有servlet对象共享一个ServletContext。

ServletConfig：由servlet容器创建，servlet容器初始化一个servlet对象时，将servletconfig对象传递给该对象

ServletRequest：由servlet容器创建，其封装了客户端向servlet对象发送的请求信息，作为servlet对象的service方法的参数 ServletResponse：由servlet容器创建，其封装了servlet对象向客户端发送的响应信息，作为servlet对象的service方法的参数

Servlet接口

public interface Servlet {
    public void init(ServletConfig config) throws ServletException;

    public ServletConfig getServletConfig();

    public void service(ServletRequest req, ServletResponse res) throws ServletException, IOException;

    public String getServletInfo();

    public void destroy();
}

servletAPI

方法名 功能 调用
init 加载servlet的配置信息（servlet类名，servlet映射等） 第一次访问servlet时调用
service 处理客户端发来的请求，并响应数据给客户端 每次发送请求时调用
destroy 销毁servlet 停止web容器（tomcat等）前调用

常用Servlet

GenericServlet

GenericServlet为Servlet接口的实现类，它实现了init、getServletConfig、getServletInfo、destroy方法，service方法为抽象方法。

HttpServlet

HttpServlet继承了GenericServlet类，实现了service方法，增加了多个请求方式对应的处理方法。httpservlet可获取客户端请求的方式，进而把请求交给对应方式的方法处理。

Servlet生命周期

Servlet对象是用户第一次访问时创建，对象创建之后就驻留在内存里面了，响应后续的请求。Servlet对象一旦被创建，init()方法就会被执行，客户端的每次请求导致service()方法被执行，Servlet对象被摧毁时(Web服务器停止后或者Web应用从服务器里删除时)，destory()方法就会被执行。

HttpServlet开发示例

通过XML开发

创建Servlet

package com.example.Servlet;

import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

public class MyHttpServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        System.out.println("=====MyHttpServlet--doPost=====");
        String name = req.getParameter("name");
        resp.getWriter().write(name);
    }

    @Override
    protected void service(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        super.service(req, resp);
    }

    @Override
    public void destroy() {
        System.out.println("=====MyHttpServlet--destroy=====");
    }

    @Override
    public void init() throws ServletException {
        System.out.println("=====MyHttpServlet--init=====");
    }
}

package com.example.Servlet;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;

import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

//注解
@WebServlet(name = "myServlet", value = "/myServlet")

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {

    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        System.out.println("=====MyHttpServlet--doPost=====");
        String name = req.getParameter("name");
        resp.getWriter().write(name);
    }

    @Override
    protected void service(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
        super.service(req, resp);
    }

    @Override
    public void destroy() {
        System.out.println("=====MyHttpServlet--destroy=====");
    }

    @Override
    public void init() throws ServletException {
        System.out.println("=====MyHttpServlet--init=====");
    }
}

Tomcat

Tomcat是一种B/S架构服务器，支持servlet规范，客户端默认采用UTF-8字符格式发数据，服务器端默认采用ISO8859-1字符格式处理数据。因此开发JavaWeb时需要考虑到字符编码问题。

tomcat目录文件夹：

conf：存放配置文件，server.xml是tomcat服务的配置文件，web.xml是项目的配置文件。

webapps：存放打包的web项目。
