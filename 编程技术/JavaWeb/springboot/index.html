Springboot

Springboot是什么

Springboot是整合Spring技术栈的一站式开发框架
Springboot是简化Spring技术栈的快速开发脚手架

为什么要用Springboot

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can “just run”.

能够快速开发出基于Spring的生产级应用

特点

  • Create stand-alone Spring applications

    创建独立的Spring应用

  • Embed Tomcat, Jetty or Undertow directly (no need to deploy WAR files)

​ 直接内嵌Tomcat，Jetty或者Undertow，无需部署war文件

  • Provide opinionated ‘starter’ dependencies to simplify your build configuration

​ 提供自定义的启动器来简化构建配置

  • Automatically configure Spring and 3rd party libraries whenever possible

​ 自动配置Spring和第三方库

  • Provide production-ready features such as metrics, health checks, and externalized configuration

​ 提供用于生产级别的功能，如指标、健康检查和外部化配置

  • Absolutely no code generation and no requirement for XML configuration

​ 完全无代码生成，无需XML配置文件

时代背景

微服务

https://martinfowler.com/microservices/

简而言之，微服务架构风格是一种将单个应用程序作为小型服务套件来开发的方法，每个服务在自己的进程中运行，并通过轻量级机制（通常是 HTTP 资源 API）进行通信。 这些服务围绕业务功能构建，可通过全自动部署机制独立部署。 这些服务可能使用不同的编程语言编写，也可能使用不同的数据存储技术。

  • 微服务是一种架构风格
  • 将一个应用拆分为一组小型服务来开发
  • 每个服务运行在自己的进程内，也就是可独立部署和升级
  • 服务之间使用轻量级HTTP交互
  • 这些服务围绕业务功能拆分
  • 可以由全自动部署机制独立部署
  • 去中心化，服务自治。服务可以使用不同的编程语言开发，使用不同的存储技术

自动配置原理

Springboot特点

依赖管理

  • spring-boot-dependencies

在maven的pom.xml配置文件中会配置一个spring-boot-starter-parent父依赖

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.7.18</version>
    <relativePath/> <!-- lookup parent from repository -->
</parent>

点进spring-boot-starter-parent会还有一个父依赖spring-boot-dependencies

<parent>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-dependencies</artifactId>
  <version>2.7.18</version>
</parent>

点进spring-boot-dependencies 其在properties标签中声明了所有开发中常用的依赖的版本号,自动进行版本仲裁机制。

![](/Users/pacemrc/Library/Application Support/typora-user-images/image-20240711161347030.png)

  • spring-boot-starter-*

springboot让整合其他框架变得十分简单，只需要引入spring-boot-starter-*依赖，这个组件的所有常规依赖就可以被自动加载

所有启动器查阅

https://docs.spring.io/spring-boot/docs/2.7.18/reference/html/using.html#using.build-systems.starters

例如

//springboot引入使用 Thymeleaf 视图构建 MVC Web 应用程序
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>
//springboot引入使用 Tomcat 作为嵌入式 servlet 容器的启动器
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-tomcat</artifactId>
</dependency>

所有以spring-boot-starter-*命名格式的依赖都是springboot官方配置的依赖，也可自定义结合第三方库配置启动器，第三方启动器的命名格式为*-spring-boot-starter

例如

<dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>3.0.3</version>
</dependency>

所有场景启动其的最底层依赖是spring-boot-starter

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter</artifactId>
  <version>2.7.18</version>
  <scope>compile</scope>
</dependency>

spring-boot-starter-parent和spring-boot-starter-*可以使得在开发过程中无需关注依赖组件的版本号，springboot会自动进行版本仲裁。

若自动仲裁的版本不符合实际开发需求，可在pom.xml文件中的properties标签中声明组件使用的版本

例如

<properties>
    <mysql.version>5.1.43</mysql.version>
</properties>

容器功能

组件添加

@Configuration

使用@Configuration注解标注在类上，表明这个类是spring的配置类。该配置类也是容器中的一个对象

@Configuration
public class MyConfig {
}
@Bean

使用@Bean注解标注在方法上，向容器内注册对象。方法名为bean对象的id，方法的返回值类型为bean对象类型，返回值，就是组件在容器中的实例。

@Bean
public User user01() {
  return new User("user01",18);
}
@Configuration(proxyBeanMethods = true)

proxyBeanMethods：代理bean的方法，默认为true，用于解决容器内组件依赖的问题

  • Full(proxyBeanMethods = true)、【保证每个@Bean方法被调用多少次返回的组件都是单实例的】
  • Lite(proxyBeanMethods = false)【每个@Bean方法被调用多少次返回的组件都是新创建的】
  • 组件依赖必须使用Full模式默认。其他默认是否Lite模式

代码示例

MyConfig.java

import ch.qos.logback.core.helpers.Transform;
import com.pacemrc.springboot.SpringbootApplication;
import com.pacemrc.springboot.domain.Pet;
import com.pacemrc.springboot.domain.User;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.condition.ConditionalOnBean;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.context.ConfigurableApplicationContext;
import org.springframework.context.annotation.*;


@ImportResource({"classpath:spring.xml"})
@Import({User.class, Transform.class})
@Configuration
/**
 * proxyBeanMethods = true  会检查容器中对象是否存在，
 * Full (proxyBeanMethods = true)
 * Lite (proxyBeanMethods = false)
 * 解决容器中组件互相依赖的问题
 *
 *
 * @Import({User.class, Pet.class})
 *   给容器中自动创建该类型的组件，调用无参构造函数，并将其注册到容器中
 */
public class MyConfig {

    @Bean
    public Pet tomcat() {
        return new Pet("tomcat");
    }

    @Bean
    @ConditionalOnBean(name = "tomcat")
    public User user01() {
        User user = new User("user01", 18);
        user.setPet(tomcat());
        return user;
    }


    public static void main(String[] args) {

        //1、返回IOC容器
        ConfigurableApplicationContext run = SpringApplication.run(SpringbootApplication.class, args);

        // 查看springboot默认所有的bean
        String[] beanDefinitionNames = run.getBeanDefinitionNames();
        for (String beanDefinitionName : beanDefinitionNames) {
            System.out.println(beanDefinitionName);
        }

        //3、从容器中获取组件
        User user01 = run.getBean("user01",User.class);
        System.out.println(user01);

        User user02 = run.getBean("user01",User.class);
        System.out.println(user02);

        System.out.println("组件是否相等：" + (user01 == user02));

        //配置类也是一个容器组件,组件是单例模式
        MyConfig config = run.getBean(MyConfig.class);
        System.out.println(config);

        //如果@Configuration(proxyBeanMethods = true)代理对象调用方法。SpringBoot总会检查这个组件是否在容器中有。
        //保持组件单实例
        User user1 = config.user01();
        User user2 = config.user01();
        System.out.println("组件是否相等：" + (user1 == user2));

        User user = run.getBean("user01", User.class);
        Pet pet = run.getBean("tomcat", Pet.class);

        System.out.println("组件是否是同一个" + (user.getPet() == pet));


        //查看指定类型的所有组件
        String[] beanNamesForType = run.getBeanNamesForType(User.class);
        for (String beanName : beanNamesForType) {
            System.out.println(beanName);
        }

        // 查看指定类型的bean,包括子类
        Transform transform = run.getBean(Transform.class);
        System.out.println(transform);

        boolean result1 = run.containsBean("tomcat");
        System.out.println("是否存在tomcat组件：" + result1);

        boolean result2 = run.containsBean("user01");
        System.out.println("是否存在user01组件：" + result2);
    }

}

@Conditional

条件装配，当满足特定条件时才向容器中注册组件

@Conditional是一个父接口，该接口实现了很多子接口

![](/Users/pacemrc/Library/Application Support/typora-user-images/image-20240711173431259.png)

该注解应用在@Bean注解的方法上，则表明当满足条件时则注册该@bean组件

该注解应用在@Configuration注解类上，则表明满足特定条件时该配置类内的所有@bean组件才会注册到容器

import com.pacemrc.springboot.SpringbootApplication;
import com.pacemrc.springboot.domain.Pet;
import com.pacemrc.springboot.domain.User;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.condition.ConditionalOnBean;
import org.springframework.context.ConfigurableApplicationContext;
import org.springframework.context.annotation.*;

@Configuration
public class MyConfig {

    @Bean
    public Pet tomcat() {
        return new Pet("tomcat");
    }

    @Bean
    @ConditionalOnBean(name = "tomcat")
    public User user01() {
        User user = new User("user01", 18);
        user.setPet(tomcat());
        return user;
    }


    public static void main(String[] args) {
        //1、返回我们IOC容器
        ConfigurableApplicationContext run = SpringApplication.run(SpringbootApplication.class, args);

        //2、查看容器里面的组件
        String[] names = run.getBeanDefinitionNames();
        for (String name : names) {
            System.out.println(name);
        }

        boolean tomcat = run.containsBean("tomcat");
        System.out.println("容器中Tomcat组件："+tomcat);

        boolean user01 = run.containsBean("user01");
        System.out.println("容器中user01组件："+user01);

        boolean tom22 = run.containsBean("tom22");
        System.out.println("容器中tom22组件："+tom22);

    }
}

从spring.xml配置文件中向容器中注册组件

spring.xml

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="user05" class="com.pacemrc.springboot.domain.User">
        <property name="name" value="John"/>
        <property name="age" value="35"/>
    </bean>

    <bean id="pig" class="com.pacemrc.springboot.domain.Pet">
        <property name="name" value="pig"/>
    </bean>
</beans>

SpringbootApplication.java

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ConfigurableApplicationContext;


//@SpringBootConfiguration
//@EnableAutoConfiguration
//@ComponentScan(value = "com.pacemrc")

/**
 * SpringBootApplication等同于开启SpringBootConfiguration、EnableAutoConfiguration、ComponentScan三个注解
 */
@SpringBootApplication
public class SpringbootApplication {

    public static void main(String[] args) {

        //1、返回IOC容器
        ConfigurableApplicationContext run = SpringApplication.run(SpringbootApplication.class, args);

        boolean user05 = run.containsBean("user05");
        System.out.println("是否存在user05组件：" + user05);

        boolean pig = run.containsBean("pig");
        System.out.println("是否存在pig组件：" + pig);
    }

}

是否存在user05组件：true
