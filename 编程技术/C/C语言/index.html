C语言基础

GCC编译

安装依赖

CentOS

yum install gcc -y
yum groupinstall "Development Tools" -y

编译方式

  • 动态链接编译
#仅生成可执行文件，不包含依赖的库，如果目标系统上缺少依赖的库文件，则会运行报错
gcc -o hello hello.c
  • 静态链接编译
#将所有库打包到可执行文件中，移植到其他系统上通常可正常运行，缺点文件体积大
gcc -static -o hello hello.c

系统兼容性的关键点

  • CPU架构

必须在相同架构（如x86_64）上运行，ARM架构无法执行x86编译的程序。

  • 内核版本

静态编译的程序可能依赖高版本内核的系统调用，在旧内核上存在崩溃的可能。

  • glibc版本

动态链接的程序要求目标系统的glibc版本小于等于编译环境的版本。glibc向低版本兼容。

  • 依赖库

ldd /usr/local/php/sbin/php-fpm

将二进制文件依赖的动态链接库复制到指定目录下打包，再上传到目标服务器的库文件目录下，如/lib64

查看二进制程序依赖的共享库：

ldd /usr/local/php/sbin/php-fpm
