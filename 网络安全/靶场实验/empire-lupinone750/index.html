empire-lupinone750

环境信息

靶场地址：https://www.vulnhub.com/entry/empire-lupinone,750/

kali攻击机：192.168.26.159

靶机：192.168.26.170(主机扫描获取)

渗透测试

信息收集

主机发现

使用kali的namp工具对同网段主机进行主机存活识别

nmap -sS -T4 192.168.26.0/24

除去同网段的网关地址（192.168.26.1、192.168.26.2、192.168.26.254）以及kali地址（192.168.26.159），发现新的主机192.168.26.170，即为靶机IP

端口服务扫描

nmap -F -sV -O 192.168.26.170

发现靶机开放22、80端口，以及web服务的的版本号，以及操作系统内核版本

目录爆破

访问靶机的http服务：http://192.168.26.170

发现只有一张图片，查看网页源代码

没有发现有效信息，使用gobuster工具对http服务进行目录爆破，爆破字典可使用SecLists的字典

./gobuster dir -u <http://192.168.26.170> -w /usr/share/wordlists/SecLists/Discovery/Web-Content/big.txt

探测存在robots.txt文件

访问该文件：http://192.168.26.170/robots.txt

robots.txt文件是个爬虫协议，规定了禁止爬取网站中的指定网页，该网站禁止爬去/~myfiles页面

访问http://192.168.26.170/~myfiles/

访问后显示“Error 404”，并无其他有效信息，查看网页源代码

提示“”，确定大致方向是正确的

继续对http://192.168.26.170/网站进行目录爆破，使用wfuzz进行fuzz过滤掉请求响应是404的payload

wfuzz -c -w /usr/share/wordlists/SecLists/Discovery/Web-Content/directory-list-2.3-big.txt -u <http://192.168.26.170/~FUZZ> |grep -v 404

fuzz一段时间后，有一个secret的payload，尝试访问http://192.168.26.170/~secret/

该页面说明该路径下存在一个隐藏的ssh私钥文件(如.aaa.txt)，可通过fasttrack字典进行爆破，朋友的名称是icex64(可能为登陆用户名)

对ssh私钥文件进行枚举

ssh私钥文件的后缀可能为txt，key，csr，pub，pem等，使用burpsuite的intruder进行fuzz

攻击类型选择，Pitchfork，将私钥文件的文件名称和文件后缀位置设置成fuzz

进入payloads选项，设置第一个fuzz位置的payload，从文件中加载字典

再设置第二个fuzz位置的payload，手动添加后缀格式即可

进入资源池选项，设置最大请求数为50

再点击右上角的Start attack

经过一段时间的爆破，成功枚举出mysecret的payload

密码爆破

访问http://192.168.26.170/~secret/.mysecret.txt

该文件内容经过编码加密，可使用https://www.dcode.fr/identification-chiffrement网站对加密方式进行识别

显示较大可能的加密方式为base58，使用http://www.metools.info/code/c74.html网站对加密数据进行解密

解密后的数据确定是ssh密钥对，将该内容保存至ssh.key文件中

使用john工具生成密码本

/usr/share/john/ssh2john.py ssh.key > ssh.txt

根据提示，使用fasttrack字典对密码本进行破解

john --wordlist=/usr/share/wordlists/fasttrack.txt ssh.txt

成功爆破出密码：P@55w0rd!

使用ssh私钥文件登陆icex64用户

chmod 600 ssh.key
ssh icex64@192.168.26.170 -i ssh.key

权限提升

查看当前路径下的文件，存在一个user.txt文件

并无发现有效flag，查看当前用户的sudo权限

sudo -l

发现可以免密登陆到arsene用户，执行/usr/bin/python3.9 /home/arsene/heist.py命令

查看/home/arsene家目录文件，存在heist.py和note.txt文件，在查看note.txt文件

需要我们对heist.py文件进行检查，查看该文件

ls -l /home/arsene/heist.py


cat /home/arsene/heist.py

发现其导入了webbrowser库，使用find命令查找该文件路径，其路径为：/usr/lib/python3.9/webbrowser.py

编辑该文件，添加下面代码

os.system("/bin/bash")

然后运行heist.py

sudo -u arsene /usr/bin/python3.9 /home/arsene/heist.py

成功登陆上arsene用户

查看arsene用户的sudo权限

可知arsene用户可免密使用root用户身份执行pip命令，可在该网址查询相关提权操作https://gtfobins.github.io/

TF=$(mktemp -d)
echo "import os; os.execl('/bin/sh', 'sh', '-c', 'sh <$(tty) >$(tty) 2>$(tty)')" > $TF/setup.py
sudo pip install $TF

成功登陆上root账户

查看/root目录下文件，发现root.txt文件，成功获取到flag

cat /root/root.txt
