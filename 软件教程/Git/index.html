Git教程

介绍

是什么

Git是免费开源的分布式版本控制工具，可以对开发项目中的文件进行版本控制。

功能

开发项目的源代码备份

多人开发同一项目

开发项目的版本切换

实现原理

将本地项目初始化为git项目后，将工作区中需要进行版本控制的文件添加到暂存区，完善代码后需要commit操作将暂存区的文件提交到本地仓库，本地仓库中的文件可以使用push操作备份到远程仓库。

基本概念

本地仓库：将本地开发项目初始化为git项目的文件夹
远程仓库：用于托管存储项目代码的平台，如gitee，github，gitlab

分支：分支是当项目的主线版本在开发过程中接到新的需求，为了使得新需求的代码不影响主线版本而使用的技术手段。git项目在创建时默认存在一个master主分支，分支可以创建多个，彼此之间互相隔离互不影响。

标签：标签是某个分支在某一时间的状态，类似于快照。常使用vX.X.X作为版本标识。

工作区：在一路径下除.git文件夹外的区域，该区域存放着源代码、配置文件等文件

暂存区：.git文件夹下的index文件。该文件记录着处于暂存区的所有文件。

版本历史信息：

.gitignore：在idea集成git中，可通过该文件排除不需要进行提交的文件

文件的状态：

权限认证

推荐使用git 凭据管理器进行授权

基本操作

本地仓库的操作

全局配置

查看全局配置

git config --global -l

配置git全局用户名称

git config  --global user.name xxxx

配置git全局邮箱地址

git config  --global user.email xxxx@qq.com

关闭ssl证书验证

git config --global http.sslVerify false

配置代理

git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

取消代理

git config --global --unset http.proxy
git config --global --unset https.proxy

创建本地仓库

git init

将在当前目录创建一个.git文件夹

将工作区文件添加到暂存区

git add <filename>   //添加单个文件
git add --all		 //添加工作区的所有文件

执行git add命令后.git文件夹内新增一个index文件，该index文件记录着暂存区的所有文件

查看暂存区文件状态

git status

将暂存区文件到提交本地仓库

git commit -m "commit message" <filename>  #将指定文件的更改提交
git commit -m "commit message" .  #将当前目录下的所有已暂存的更改提交

查看git操作记录

git log

执行git commit操作后会自动生成一个commit id，执行git log命令会显示该ID，另外打印出提交人的信息和提交时间，（HEAD -> master）表示当前处于该版本

版本回退

git log   //获取commitID
git reset --soft commitID   //只进行对commit操作的回退，不影响工作区的文件，当发现之前的提交有错误的代码，推荐使用soft进行处理，这样会把错误的提交日志进行废弃，git log就查看不到那一次错误的提交日志。
git reset --hard commitID   //回退commit操作和工作区的文件

远程仓库的操作

创建远程仓库

以github演示，在个人主页点击new repository进入创建页面，需要定义仓库的名称，仓库的访问权限公开or私有其他默认即可，最后点击create repository完成创建。

本地仓库绑定远程仓库

git remote add testGit01 https://github.com/pacemrc/testGit.git

查看远程仓库

git remote
git remote -v

将本地仓库的代码推送到远程仓库

git push -u 远程仓库别名 分支名称
git push -f -u 远程仓库别名 分支名称    #强制覆盖之前的commit提交代码

第一次推送需要进行仓库的授权认证

将远程仓库克隆到本地

git clone https://github.com/pacemrc/testGit.git

从远程仓库拉取新的代码

git pull

分支的操作

查看分支

查看本地分支

git branch

查看远程分支

git branch -r

查看本地和远程分支

git branch -a

创建分支

git branch 分支名称
git checkout -b 分支名称  //创建并切换到新的分支

删除分支

git branch -d 分支名称

修改分支名称

git branch -M oldName newName

git branch --show-current  查看当前分支名称

切换分支

git checkout 分支名称

合并分支代码

将指定分支的代码合并到当前分支上

git merge 分支名称

标签的操作

查看标签

git tag

查看当前所在标签

git log

创建标签

git tag [tagName]

删除标签

git tag -d [tagName]

集成IDEA

IDEA配置Git

进入到All Settings–>Version Control–>Git，在Path to Git executable设置Git可执行文件的路径，点击test当出现Git的版本即成功配置

初始化本地仓库

点击菜单栏中VCS–>Create Git Repository，选择需要初始化的文件夹，将当前项目初始化为git项目

文件名颜色含义

红色表示文件未添加到工作区

绿色表示文件处于工作区

黄色表示文件不会参与提交

白色（无颜色）表示文件已提交至版本控制

.gitignore

排除项目中需要添加到git暂存区的文件，需手动创建。排除的文件通常是/target/目录，*.log等文件

从远程仓库克隆

点击菜单栏中Git–>Clone

在Repository URL中，Version control选择Git，URL输入远程仓库的地址，Directory输入在本地创建的地址

将本地仓库推送到远程仓库

如何取消Git项目

进入Settings–>Version Control–>Directory Mappings设置，将项目的映射关系删除，同时将项目文件夹下的.git文件夹删除掉
