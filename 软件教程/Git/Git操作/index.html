Git操作

权限认证

推荐使用git 凭据管理器进行授权

本地仓库

查看全局配置

git config --global -l

配置git全局用户名称

git config  --global user.name xxxx

配置git全局邮箱地址

git config  --global user.email xxxx@qq.com

关闭ssl证书验证

git config --global http.sslVerify false

配置代理

git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

取消代理

git config --global --unset http.proxy
git config --global --unset https.proxy

创建本地仓库

git init
#在当前目录创建一个.git文件夹

将工作区文件添加到暂存区

git add <filename>   #添加单个文件
git add --all		 #添加工作区的所有文件
执行git add命令后.git文件夹内新增一个index文件，该index文件记录着暂存区的所有文件

查看暂存区文件状态

git status

将暂存区文件到提交本地仓库

git commit -m "commit message" <filename>  #将指定文件的更改提交
git commit -m "commit message"  #将当前目录下的所有已暂存的更改提交

查看提交信息

git log
#执行git commit操作后会自动生成一个commit id，执行git log命令会显示该ID，另外打印出提交人的信息和提交时间，（HEAD -> master）表示当前处于该版本

版本回退

git log   #获取commitID
git log --oneline
git reset --soft commitID   #只进行对commit操作的回退，不影响工作区的文件
git reset --hard commitID   #回退commit操作和工作区的文件

#会把未提交的文件强制删除
git clean -fd

远程仓库

创建远程仓库

Github/Gitlab上创建

本地仓库绑定远程仓库

git remote add 远程仓库别名 https://github.com/pacemrc/testGit.git

查看远程仓库

git remote
git remote -v

将本地仓库的代码推送到远程仓库

git push -u 远程仓库别名 分支名称
git push -f -u 远程仓库别名 分支名称    #强制覆盖之前的commit提交代码

将远程仓库克隆到本地

git clone https://github.com/pacemrc/testGit.git

从远程仓库拉取新的代码

git pull

分支

查看分支

#查看本地分支
git branch

#查看远程分支
git branch -r

#查看所有分支
git branch -a

创建分支

git branch <branch_name>
git checkout -b <branch_name>  #创建并切换到新的分支

删除分支

git branch -d <branch_name>

修改分支名称

git branch -M oldName newName

git branch --show-current  #查看当前分支名称

切换分支

git checkout <branch_name>

合并分支代码

将指定分支的代码合并到当前分支上

git merge <branch_name>

标签

查看标签

git tag

查看当前所在标签

git log

创建标签

git tag <tagName>

删除标签

git tag -d <tagName>

提交标签

git push <remote_alais> <tagName>
