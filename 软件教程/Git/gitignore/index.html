.gitignore

排除项目中需要添加到git暂存区的文件，需手动创建。排除的文件通常是/target/目录，*.log等文件

规则符号 含义 示例
# 注释（不会生效，用于说明规则） # 忽略日志文件
* 通配符，匹配任意字符（0 个或多个） *.log 忽略所有.log 文件
? 匹配单个字符 test?.txt 匹配 test1.txt、test2.txt
/ 路径分隔符，用于限定目录（不加则匹配所有目录下的文件） dist/ 忽略根目录的 dist 文件夹
! 例外规则（覆盖前面的忽略，即 “不忽略这个文件 / 文件夹”） !important.log 不忽略 important.log
** 匹配任意层级的目录 **/node_modules 忽略所有目录下的 node_modules

通用.gitignore模板

# 1. 注释：说明规则用途
# 忽略操作系统自动生成的文件
.DS_Store                # Mac系统
Thumbs.db                # Windows系统
desktop.ini              # Windows系统

# 2. 忽略编辑器/IDE配置文件
.idea/                   # IntelliJ/IDEA
.vscode/                 # VS Code
*.sublime-project        # Sublime
*.swp                    # Vim临时文件
*.bak                    # 通用备份文件

# 3. 忽略编程语言/框架的依赖/构建文件
# Python
__pycache__/             # Python编译后的缓存
*.pyc                    # Python字节码文件
venv/                    # 虚拟环境
.env                     # 环境变量文件（敏感信息）
requirements.txt~        # 临时文件

# JavaScript/Node.js
node_modules/            # Node依赖
dist/                    # 打包后的构建目录
build/                   # 构建目录
npm-debug.log            # npm日志
yarn.lock                # 可选：如果团队统一用npm，可忽略yarn.lock

# Java
target/                  # Maven/Gradle构建目录
*.class                  # Java编译文件

# 4. 忽略日志/临时文件
logs/                    # 日志文件夹
*.log                    # 所有日志文件
tmp/                     # 临时文件夹
*.tmp                    # 临时文件

# 5. 例外规则：不忽略特定文件
!important.log           # 即使上面忽略了*.log，也保留important.log
!src/utils/*.js          # 不忽略src/utils下的所有js文件
