OpenClaw

安装

安装命令

npm install -g openclaw@latest

配置

运行配置向导

openclaw onboard --install-daemon

常用操作

启动网关

前台启动：
openclaw gateway run
后台启动：
openclaw gateway start

获取带令牌的仪表盘URL：

openclaw dashboard --no-open

更新版本

openclaw update

诊断

openclaw doctor

查看版本

openclaw -V

插件

微信插件

安装插件：

npx -y @tencent-weixin/openclaw-weixin-cli@latest install
openclaw plugins install "@tencent-weixin/openclaw-weixin@latest"

登录认证：

openclaw channels login --channel openclaw-weixin

重启服务：

openclaw gateway restart

卸载插件：

openclaw plugins uninstall @tencent-weixin/openclaw-weixin

若 CLI 异常或卸载失败，可手动执行：

  1. 禁用插件：openclaw plugins disable @tencent-weixin/openclaw-weixin
  2. 删除目录：rm -rf ~/.openclaw/extensions/@tencent-weixin/openclaw-weixin
  3. 清配置：编辑 ~/.openclaw/openclaw.json，删除 plugins.entries 和 plugins.installs 中该插件的条目
  4. 执行openclaw doctor --fix检查

蓝信插件

安装插件：

openclaw plugins install @lansenger/openclaw-channel-lansenger@latest

绑定机器人：

openclaw channels add --channel Lansenger --token '{AppID}:{AppSecret}:https://apigw.lx.qianxin.com'

重启服务：

openclaw gateway restart

卸载插件：

openclaw plugins uninstall @lansenger/openclaw-channel-lansenger

故障问题

  1. 问题现象：
    origin not allowed (open the Control UI from the gateway host or allow it in gateway.controlUi.allowedOrigins)
    解决：
    允许非安全认证（解决跨域问题）

    openclaw config set gateway.controlUi.allowInsecureAuth true

    允许指定域名连接

    openclaw config set gateway.controlUi.allowedOrigins '["http://10.58.120.200:18789"]'

  2. 问题现象：
    control ui requires device identity (use HTTPS or localhost secure context)
    解决：
    禁用设备身份验证（允许 HTTP 访问）

    openclaw config set gateway.controlUi.dangerouslyDisableDeviceAuth true
