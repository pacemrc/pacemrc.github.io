curl

curl

语法

curl [option] [url]

用法

查看网页源码

curl www.baidu.com

User Agent

  • -A

指定客户端的用户代理标头，即User-Agent

curl -A “Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko)” http://192.168.220.169

curl 的默认用户代理字符串是curl/[version]

去掉User-Agent标头

curl -A “” www.baidu.com

curl -H ‘User-Agent: php/1.0’ www.baidu.com

Cookie

  • -b

参数用来向服务器发送 Cookie。

curl -b “key1=value1” http://192.168.220.169

上面命令会生成一个标头Cookie: key1=value1，向服务器发送一个名为key1、值为value1的 Cookie。

curl -b “key1=value1;key2=value2” http://192.168.220.169

发送两个Cookie

curl -b cookies.txt http://192.168.220.169

读取本地文件cookies.txt，里面是服务器设置的 Cookie。

读取cookie文件

  • -c

参数将服务器设置的 Cookie 写入一个文件。

curl -c cookies2.txt http://192.168.220.169

传输请求实体数据

  • -d

参数用于发送 POST 请求的数据体。

curl -d “login=caofalin & pasword=123456” -X POST http://192.168.220.169

curl -d “login=caofalin” -d “pasword=123456” -X POST http://192.168.220.169

使用-d参数以后，HTTP 请求会自动加上标头Content-Type : application/x-www-form-urlencoded。并且会自动将请求转为 POST 方法，因此可以省略-X POST。

  • d参数可以读取本地文本文件的数据，向服务器发送。

curl -d “@data.txt” http://192.168.220.169

传输实体数据编码

  • -data-urlencode

参数等同于-d，发送 POST 请求的数据体，区别在于会自动将发送的数据进行 URL 编码。

curl –data-urlencode “comment=hello world” http://192.168.220.169

Referer

  • -e

参数用来设置 HTTP 的标头Referer，表示请求的来源。

curl -e “https://www.baidu.com?q=baidu“ http://192.168.220.169

文件传输

  • -F

向服务器上传二进制文件

$ curl -F ‘file=@photo.png‘ https://google.com/profile

上面命令会给 HTTP 请求加上标头Content-Type: multipart/form-data，然后将文件photo.png作为file字段上传。

指定 MIME 类型

$ curl -F ‘file=@photo.png;type=image/png’ https://google.com/profile

上面命令指定 MIME 类型为image/png，否则 curl 会把 MIME 类型设为application/octet-stream。

指定文件名

$ curl -F ‘file=@photo.png;filename=me.png’ https://google.com/profile

上面命令中，原始文件名为photo.png，但是服务器接收到的文件名为me.png。

字符串查询

  • -G

参数用来构造 URL 的查询字符串。

curl -G -d “q=qianxin & count=2” http://192.168.220.169

上面命令会发出一个 GET 请求，实际请求的 URL 为

[https://google.com/search?q=kitties&count=20。![](https://google.com/search?q=kitties&count=20%E3%80%82))

如果数据需要 URL 编码，可以结合–data–urlencode参数

curl -G -data-urlencode “comment=hello world” http://192.168.220.169

添加请求头字段

  • -H

参数添加 HTTP 请求的标头。

curl -H “Accept-Language: en-US” http://192.168.220.169

curl -H “Accept-Language: en-US” -H “Secret-Message: xyzzy” http://192.168.220.169

curl -d “{“login”: “emma”, “pass”: “123”}” -H “Content-Type: application/json” http://192.168.220.169

显示头信息

  • -i

参数打印出服务器回应的 HTTP 标头。

curl -i http://192.168.220.169

head请求并打印

  • -I

参数向服务器发出 HEAD 请求，然会将服务器返回的 HTTP 标头打印出来。

curl -I http://192.168.220.169

  • -head参数等同于-I。

跳过 SSL 检测

  • -k

参数指定跳过 SSL 检测,不会检查服务器的 SSL 证书是否正确。

curl -k https://www.baidu.com

自动跳转

  • -L

参数会让 HTTP 请求跟随服务器的重定向。curl 默认不跟随重定向。

curl -L -d ‘tweet=hi’ https://api.twitter.com/tweet

限制带宽

  • -limit-rate

用来限制 HTTP 请求和回应的带宽，模拟慢网速的环境。

curl –limit-rate 200k https://google.com

将带宽限制在每秒 200K 字节。

文件下载

  • -o

参数将服务器的回应保存成文件，等同于wget命令。

curl -o apache.html http://192.168.220.169

  • s参数将不输出错误和进度信息。

curl -s -o apache.html http://192.168.220.169

输出错误信息

  • -S

参数指定只输出错误信息，通常与-s一起使用。

curl -s -o /dev/null https://google.com

身份认证

  • -u

参数用来设置服务器认证的用户名和密码。

curl -u ‘bob:12345’ http://192.168.220.169

上面命令设置用户名为bob，密码为12345，然后将其转为 HTTP 标头Authorization: Basic Ym9iOjEyMzQ1。

curl 能够识别 URL 里面的用户名和密码。

curl http://bob:12345@http://192.168.220.169

上面命令能够识别 URL 里面的用户名和密码，将其转为上个例子里面的 HTTP 标头。

curl -u ‘bob’ https://google.com/login

只设置了用户名，执行后，curl 会提示用户输入密码。

显示通信过程

  • -v

参数输出通信的整个过程，用于调试。

curl -v http://192.168.220.169

指定请求代理

  • -x

参数指定 HTTP 请求的代理。

curl -x socks5://james:cats@myproxy.com:8080 https://www.example.com

上面命令指定 HTTP 请求通过myproxy.com:8080的 socks5 代理发出。

如果没有指定代理协议，默认为 HTTP。

$ curl -x james:cats@myproxy.com:8080 https://www.example.com

上面命令中，请求的代理使用 HTTP 协议。

请求方法

  • -X

参数指定 HTTP 请求的方法。

curl -X head http://192.168.220.169

curl -X POST http://192.168.220.169

curl -X GET http://192.168.220.169
