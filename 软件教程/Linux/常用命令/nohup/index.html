nohup
功能
用于在系统后台不挂断地运行命令，退出终端不会影响程序的运行。
nohup命令，在默认情况下（未指定重定向时），会输出一个名叫nohup.out的文件到当前目录下，如果当前的nohup.out文件不可写，则输出重定向到$HOME/nohup.out文件中。
语法
nohup Command [Arg …] [ & ]
Command：要执行的命令。
Arg：一些参数，可以指定输出文件。
&：让命令在后台执行，终端退出后仍然执行。
eg.
在后台执行 root 目录下的 runoob.sh 脚本，并重定向输入到 runoob.log 文件：
nohup /root/runoob.sh > runoob.log 2>&1 &
如何停止后台运行程序
ps -aux | grep “runoob.sh“
a : 显示所有程序
u : 以用户为主的格式来显示
x : 显示所有程序，不区分终端机

kill -9 进程号PID
2>&1
将标准错误（2）重定向到标准输出（&1），标准输出（&1）再被重定向输入到runoob.log文件中。
0 - stdin（standard input，标准输入）
1 - stdout （standard output，标准输出）
2 - stderr（standard error，标准错误输出）
