Windows常见系统进程

核心系统进程

这些进程无法结束，是操作系统的基础。是 Windows 系统运行的根本，如果它们意外终止，会导致系统蓝屏或重启。

进程名 (Image Name) 文件名 (File Name) 功能作用描述
System ntoskrnl.exe Windows 操作系统内核。它代表操作系统本身的核心部分，运行在核心态（Kernel Mode）。它负责管理硬件抽象、内存管理、进程和线程调度、安全等最底层的任务。注意： 在“进程”选项卡中看到的“System”进程并不是一个用户态进程，而是内核本身所占用的资源（如CPU时间）的占位符。
System Idle Process N/A (系统虚拟进程) 系统空闲进程。它不是一个真正的进程，而是用于衡量CPU空闲程度的计数器。当CPU无事可做时，就会执行“System Idle Process”，其显示的CPU占用率即为当前CPU的空闲率。该进程的PID始终为0。
svchost.exe svchost.exe Service Host Process。这是最重要的进程之一。为了减少系统资源消耗，微软将多个系统服务打包，共享一个svchost.exe进程。因此，你会在任务管理器中看到多个svchost.exe实例。每个实例托管一个或多个系统服务（如Windows Update、网络相关服务、事件日志等）。
csrss.exe csrss.exe Client/Server Runtime Subsystem。这是一个关键的用户态子系统，负责控制台窗口、线程创建/销毁以及部分16位虚拟MS-DOS环境。在较新版本的Windows中，其功能已被大幅削减，但它仍然是核心进程。
wininit.exe wininit.exe Windows Initialization Process。系统启动时最早创建的进程之一，它以SYSTEM权限运行，并负责启动一些关键的系统服务，例如： • services.exe（服务控制管理器，SCM） • lsass.exe（本地安全认证子系统服务） • lsm.exe（本地会话管理器服务）
services.exe services.exe **Service Control Manager (SCM)**。这是Windows服务的总管家。它负责启动、停止系统服务，并管理服务之间的依赖关系。所有通过“服务”管理单元管理的项目都由它控制。
lsass.exe lsass.exe Local Security Authority Subsystem Service。安全核心进程，负责系统的安全策略、用户认证（例如域登录、本地登录）、审计日志以及生成安全令牌。注意： 该进程是黑客和恶意软件（如Mimikatz）的重点攻击目标，因为它内存中可能存有密码哈希或凭证。
winlogon.exe winlogon.exe Windows Logon Application。管理用户登录和注销。当您按下Ctrl+Alt+Delete时，由它负责激活安全桌面（Secure Desktop）并加载登录界面。它与lsass.exe紧密协作完成认证过程。
Explorer.exe explorer.exe Windows Explorer。这提供了图形用户界面（GUI），包括任务栏、桌面、文件资源管理器等。在服务器核心版（Server Core）安装中，默认没有此进程，因为那是一个没有图形界面的命令行环境。

网络与安全相关进程

进程名 (Image Name) 文件名 (File Name) 功能作用描述
spoolsv.exe spoolsv.exe Print Spooler Service。管理所有打印作业队列和所有与打印相关的操作。如果服务器不充当打印服务器，可以考虑禁用此服务。
svchost.exe (网络相关实例) svchost.exe 如前所述，svchost.exe托管了许多服务，包括至关重要的网络服务，例如： • DHCP Client / DNS Client • Network Connections • Windows Firewall (mpssvc) • HTTP Service (如果安装了IIS)
taskhost.exe / taskhostw.exe taskhost*.exe Host Process for Windows Tasks。类似于svchost.exe，但它专门用于托管DLL形式的任务（如计划任务、UI相关任务），而不是服务。

管理工具与应用程序进程

这些进程通常在你运行特定管理工具时出现。

进程名 (Image Name) 文件名 (File Name) 功能作用描述
mmc.exe mmc.exe Microsoft Management Console。当你打开“服务器管理器”、“事件查看器”、“磁盘管理”或任何其他.msc后缀的管理控制台单元时，都会运行此进程。它是这些管理工具的宿主外壳。
ServerManager.exe ServerManager.exe 服务器管理器。这是Windows Server中用于添加/删除角色和功能、管理已安装服务器角色等的图形化主界面工具。
powershell.exe / cmd.exe powershell.exe/ cmd.exe 命令行界面。分别是PowerShell和命令提示符（CMD）的解释器进程。
dllhost.exe dllhost.exe COM Surrogate。用于承载和执行COM对象（Component Object Model），这些对象本身不是可执行文件（.exe），而是动态链接库（.dll）。例如，一些 shell 扩展或后台处理任务会使用它。
