基于Hexo+Github Actions方案搭建Github博客

部署方案

  1. 纯静态网页文件部署

创建一个公开网页仓库，仅将网站的静态网页文件上传到该仓库。

优点：搭建快速，安全，网站源码他人不可见。缺点：网站源码在本地，不易维护。

  1. 网站源码和静态网页文件同一仓库部署

创建一个公开网页仓库，将网站的Hexo源码和静态网页文件上传到Github该仓库。

优点：网站源码易维护，不易丢失；缺点：网站仓库需为公开状态，其他人可直接访问到网站源码，不安全。

  1. 网站源码和静态网页文件仓库分离部署

分别创建一个公开网页仓库和私有源码仓库，公开网页仓库存放网站静态文件，私有源码仓库存放网站源码。

优点：网站易维护，安全性高。

本文讨论的是第三种部署方案。

整体流程

使用hexo在本地创建一个网站目录，网站文章的创建编写等操作均在本地。文章创作完成后，使用git创建为本地仓库，使用git将网页源代码文件提交到远程私有源码仓库，私有源码仓库配置了Github Actions工作流，当监听到有代码提交操作时会自动触发工作流将网站源代码编译生成网页静态文件，之后私有源码仓库将静态网页文件提交到公开网页仓库，公开网页仓库使用了Github Pages技术当监听到代码提交时会自动将静态网页文件部署到内部web容器中，最后可通过域名访问。

部署过程

安装软件

安装Git、Node.js、Hexo。参考：https://hexo.io/zh-cn/docs/

生成访问Token

Github远程仓库的访问需要进行身份认证，认证的方式有Person access token、SSH keys等。此处使用token方式进行仓库的权限认证。在Settings-->Developer Settings-->Personal access tokens-->Token（classic）中生成一个token。该Token用于私有源码仓库的工作流中定义的yml脚本环境访问公开网页仓库。

设置Token名称，有效期选择No expiration，权限范围选择repo和workflow即可，最后点击生成token。生成的token仅出现一次，后续无法查看，需保存下来。

创建私有源码仓库

创建一个仓库存放网站源代码，可视性设置Private

配置隐私变量

在私有源码仓库的Settings-->Secrets and variables-->Actions中，设置Name为DEPLOY_TOKEN，Secret设置生成的token值。该变量会在工作流的yml文件中引用。

创建公开网站仓库

创建网站仓库，仓库名格式为username.github.io，公开性设置为Public

创建本地网站

创建Hexo项目

hexo init blog

执行该命令将在当前目录下创建一个blog文件夹，从 https://github.com/hexojs/hexo-starter.git仓库拉取文件，并自动下载相关依赖包。

github将默认分支名已从master修改为main，安装git客户端时尽量选择重写主分支名，或者执行git branch -M master main将主分支名修改为main

创建文章

hexo更多操作查看https://hexo.io/zh-cn/docs/commands 本文更多演示博客搭建效果

hexo new beijing

更换主题

下载主题

笔者使用的主题：https://github.com/fluid-dev/hexo-theme-fluid.git

git clone -b master https://github.com/fluid-dev/hexo-theme-fluid.git themes/fluid

执行命令将在themes文件夹下创建fluid文件夹

应用主题

  1. 修改网站目录下的_config.yml配置文件，将theme字段值修改为theme: fluid

  1. 设置主题文件。在themes/fluid主题文件夹下，将_config.yml文件重命名为_config.fluid.yml并复制到网站主目录下。

创建Github Actions工作流

在网站主目录.github文件夹下创建workflows文件夹，进入并创建一个autodeploy.yml文件，该文件定义了一些列动作，主要为当网站源码仓库监听到push操作后即自动生成静态网页文件并将静态文件推送到网站仓库。

注意：

  1. 自行替换pacemrc关键字
  2. 自行确认项目的主分支名称，本配置使用的是main，若为master需将main替换为master
  3. 文件中$的变量名为上述步骤中设置的变量名

autodeploy.yml

name: AutoDeploy

on:
  push:
    branches:
      - main  #需确认分支名称

  release:
    types:
      - published

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout branch
      uses: actions/checkout@v3
      with:
        ref: main

    - name: Install NodeJS
      uses: actions/setup-node@v3
      with:
        node-version: "20.x"

    - name: Install Hexo
      run: |
        export TZ='Asia/Shanghai'
        npm install hexo-cli -g

    - name: Cache Hexo
      uses: actions/cache@v3
      id: cache
      with:
        path: node_modules
        key: ${{runner.OS}}-${{hashFiles('**/package-lock.json')}}

    - name: Install Dependency
      if: steps.cache.outputs.cache-hit != 'true'
      run: |
        npm install --save

    - name: Generate static files
      run: |
        hexo clean
        hexo generate

    - name: Push static file to website repo
      run: |
        cd ./public
        git init
        git config --global user.name pacemrc
        git config --global user.email xxxxx@qq.com
        git branch -M master main
        git add .
        git commit -m "$(date +"%Z %Y-%m-%d %A %H:%M:%S") Updated By Github Actions"
        git push --force --quiet "https://pacemrc:${{ secrets.DEPLOY_TOKEN }}@github.com/pacemrc/pacemrc.github.io.git" main:main #需确认分支名称

提交网站源码

  1. 删除主题文件夹themes\fluid\下的.git文件夹

  1. 将源码网站创建为Git仓库
git init
git add --all
git commit -a -m "first commit"
git remote add blog https://github.com/pacemrc/blog.git
git push -u blog main

验证

当私有源码仓库监听到代码提交操作时，自动触发定义的工作流动作。

同时公开网站仓库监听到代码提交操作时，也会自动触发预置的pages-build-deployment工作流操作。

hexo new shanghai   #创建一个标题为shanghai的文章
git add --all
