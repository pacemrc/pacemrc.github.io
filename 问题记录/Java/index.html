运行java类引入第三方jar包依赖

场景：

1、在Linux系统上编写了个java文件，该java文件引入了jdk之外的第三方类，需要怎么编译？

javac -cp/-classpath jar_file_path test.java
javac -classpath lib/hadoop-common-3.3.1.jar test.java
javac -cp lib/hadoop-common-3.3.1.jar test.java

2、在成功编译java文件后，java运行该class文件，提示找不到其他的第三方类，发现第一个第三方jar文件的类还依赖其他类。现已把所需的所有依赖jar包存放在一个路径，如何使用java命令在运行时去搜索指定路径下的jar包里的类？

使用Extension Classloader来加载

java -Djava.ext.dirs=$JAVA_HOME/jre/lib/ext:jar_file_dir_path test
java -Djava.ext.dirs=$JAVA_HOME/jre/lib:/root/test/lib test

3、Linux上使用javac命令指定多个依赖jar包编译java文件

javac -cp path/jar1.jar:path/jar2.jar test.java

自定义生成Jre

从Java9开始Java开始不再提供JRE，需要自己手动生成。JLink功能的主要目的是创建我们自己的定制JRE

指定模块生成JRE

模块名为jmods目录下的文件前缀

bin/jlink --module-path jmods --add-modules moduleName1[,moduleName2,...] --compress=2 --output jre

所有模块生成JRE

bin/jlink --module-path jmods --add-modules ALL-MODULE-PATH --compress=2 --output jre

查看Jar包依赖的JDK内置模块

bin\jdeps.exe --print-module-deps --ignore-missing-deps D:\javaProjects\JowtoTool\target\JowtoTool.jar
